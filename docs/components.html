The Scarlet Banner

Components

Link

Link
Markup <a href="#">Link</a>
  • Links go places, while buttons do things. The difference is that buttons change the state of the website. That can be on the server side or the client side, but a button implies that some action is being taken, and something, somewhere is different now. Links take you somewhere else, but neither the place you were nor the place you’re going have changed when you do so.
  • Yes, buttons are routinely abused to work as just bigger, more prominent links — usually along with other bad design decisions that visually mark almost all of the items on a page as the most important thing, creating frustration and increasing the user’s cognitive load. It’s a terrible practice, and just because it’s common is no reason to do it.
  • In places where you might be tempted to use a button to go somewhere because the link should be visually prominent, consider using the prominent link component instead.
  • If you need to present the user with a prominent decision point when hen could go to any of several different places, and each one calls for some significant explanation, you might consider the panel link component.

Prominent Link

Left-Aligned Link Center-Aligned Link Right-Aligned Link
Markup <a href="#" class="prominent">Left-Aligned Link</a> <a href="#" class="prominent center">Center-Aligned Link</a> <a href="#" class="prominent right">Right-Aligned Link</a>
  • Use this component when you want to give users the opportunity to go somewhere (meaning that they’re not changing the website back-end or front-end), but the link should be visually prominent.
  • This component is suitable as a call to action where the action is to go somewhere, rather than to do something. As with other calls to action, you should not have more than one on a page. The user can only do one thing; if you ask hen to do several things, you only create confusion and frustration. If you have multiple links to share in a sequence (e.g., a menu that takes up a significant part of a page’s content), consider using the panel link component.
  • Like any other option that creates visual prominence, do not overuse this component. Generally speaking, you should not use this more than once on a page. Visual hierarchy helps create clarity, unless you try to make everything prominent, then there is no hierarchy and no clarity, because everything is equally prominent. It’s the design equivalent of writing in all caps or shouting at someone.
  • If the call to action does do something (changing the state of the website, either on the front-end or the back-end), then you should use a button instead.

Panel Link

You can use any block-level elements you like inside the panel, including paragraphs, lists, and headings.

</a>
  • This component is best used for a menu that takes up most or all of a page (like the homepage for this design system).
  • The panel is designed to work with secondary, tertiary, and quarternary headings (<h2>, <h3> and <h4>), but not primary headings (<h1>). Each page should have only one primary heading, so if you’re considering using it in a panel, you might be using the wrong component.
  • Since the entire panel is a link, don’t use any other links or buttons inside the panel.
